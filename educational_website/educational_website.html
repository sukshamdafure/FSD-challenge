📖 Introduction to Data Structures & Algorithms

Data Structures and Algorithms (DSA) form the foundation of computer science and software engineering. Understanding DSA is crucial for writing efficient, scalable, and maintainable code. Whether you're preparing for technical interviews, building complex systems, or simply want to become a better programmer, mastering DSA is essential.

🎯 What are Data Structures?

A data structure is a specialized format for organizing, processing, storing, and retrieving data. Different data structures are suited for different kinds of applications, and some are highly specialized for specific tasks.

Think of data structures as containers that hold your data in a specific format. Just as you use different containers in real life (boxes, bottles, bags) for different purposes, you use different data structures for different computational needs.

⚙️ What are Algorithms?

An algorithm is a step-by-step procedure or formula for solving a problem. It's a sequence of well-defined instructions that takes some input, processes it, and produces an output.

Algorithms are the recipes for data manipulation. They tell you exactly what operations to perform on your data structures to achieve a desired result.

🔑 Why Learn DSA?

  • 💼 Technical Interviews: Most tech companies (Google, Amazon, Microsoft, etc.) heavily test DSA knowledge in their hiring process
  • ⚡ Performance Optimization: Understanding DSA helps you write faster, more efficient code that uses less memory
  • 🧠 Problem-Solving Skills: DSA trains your brain to think logically and break down complex problems
  • 🏗️ System Design: Essential for building scalable applications and understanding how software systems work
  • 📈 Career Growth: Opens doors to senior engineering roles and competitive programming

⏱️ Time & Space Complexity Analysis

Complexity analysis is the process of determining how the resource usage (time and memory) of an algorithm grows as the input size increases. This is crucial for comparing algorithms and choosing the most efficient solution.

📊 Big O Notation

Big O notation describes the upper bound of an algorithm's growth rate. It provides a worst-case guarantee on the resources an algorithm will consume. We focus on the dominant term and ignore constants and lower-order terms.

Common Time Complexities (Best to Worst):

Notation Name Example Description
O(1) Constant Array access Time doesn't depend on input size
O(log n) Logarithmic Binary search Divides problem in half each step
O(n) Linear Linear search Processes each element once
O(n log n) Linearithmic Merge sort Efficient sorting algorithms
O(n²) Quadratic Bubble sort Nested loops over input
O(n³) Cubic Triple nested loops Three nested iterations
O(2ⁿ) Exponential Fibonacci recursion Doubles with each input increase
O(n!) Factorial Permutations Generating all permutations

💻 Code Examples

O(1) - Constant Time

C++
// Accessing array element by index int getElement(int arr[], int index) { return arr[index]; // O(1) - single operation } // Hash table lookup int getValue(unordered_map<int, int> map, int key) { return map[key]; // O(1) average case }

O(n) - Linear Time

C++
// Finding maximum element in array int findMax(int arr[], int n) { int maxVal = arr[0]; for (int i = 1; i < n; i++) { if (arr[i] > maxVal) { maxVal = arr[i]; } } return maxVal; // O(n) - iterates through all elements }

O(n²) - Quadratic Time

C++
// Bubble Sort - nested loops void bubbleSort(int arr[], int n) { for (int i = 0; i < n-1; i++) { // O(n) for (int j = 0; j < n-i-1; j++) { // O(n) if (arr[j] > arr[j+1]) { swap(arr[j], arr[j+1]); } } } // Total: O(n²) }

O(log n) - Logarithmic Time

C++
// Binary Search - divides search space in half int binarySearch(int arr[], int n, int target) { int left = 0, right = n - 1; while (left <= right) { int mid = left + (right - left) / 2; if (arr[mid] == target) return mid; if (arr[mid] < target) left = mid + 1; else right = mid - 1; } return -1; // O(log n) }

💾 Space Complexity

Space complexity measures the total amount of memory an algorithm needs relative to the input size. It includes both auxiliary space (extra space used by the algorithm) and space used by the input.

Space Complexity Examples:

C++
// O(1) Space - only uses fixed variables int sum(int arr[], int n) { int total = 0; // Single variable for (int i = 0; i < n; i++) { total += arr[i]; } return total; // Space: O(1) } // O(n) Space - creates new array int* duplicate(int arr[], int n) { int* newArr = new int[n]; // Array of size n for (int i = 0; i < n; i++) { newArr[i] = arr[i]; } return newArr; // Space: O(n) }

🎯 Key Rules for Analysis

  • 1️⃣ Drop Constants: O(2n) becomes O(n), O(3) becomes O(1)
  • 2️⃣ Drop Lower Terms: O(n² + n) becomes O(n²)
  • 3️⃣ Different Inputs Use Different Variables: O(a + b) not O(n)
  • 4️⃣ Sequential Steps Add: O(a) then O(b) = O(a + b)
  • 5️⃣ Nested Steps Multiply: O(a) nested in O(b) = O(a × b)

💡 Pro Tip

When analyzing code, focus on loops and recursive calls. A single loop is O(n), nested loops are typically O(n²), and recursion that branches requires careful analysis of the recurrence relation. Always consider the worst-case scenario unless specifically analyzing average or best case.

📊 Arrays - Foundation of Data Structures

An array is a collection of elements stored in contiguous memory locations. It's the most fundamental and widely used data structure, providing efficient random access to elements through indexing. Arrays form the basis for many other data structures and algorithms.

🎯 What is an Array?

An array is a linear data structure where elements of the same data type are stored in adjacent memory locations. Each element can be accessed directly using its index, which starts from 0 in most programming languages.

✨ Key Characteristics

  • 📏 Fixed Size: Traditional arrays have a predetermined size that cannot change after creation (dynamic arrays like vectors/ArrayList can grow)
  • 🔢 Homogeneous: All elements must be of the same data type
  • 🎯 Random Access: Any element can be accessed in O(1) time using its index
  • 💾 Contiguous Memory: Elements are stored in adjacent memory locations, making cache-friendly
  • 0️⃣ Zero-Based Indexing: First element is at index 0, last at (size-1)

💻 Array Declaration & Initialization

C/C++ Arrays

C++
// Declaration int arr1[5]; // Uninitialized array of size 5 // Declaration with initialization int arr2[5] = {10, 20, 30, 40, 50}; // Size inferred from initializer int arr3[] = {1, 2, 3, 4}; // Size = 4 // Partial initialization (rest filled with 0) int arr4[5] = {1, 2}; // {1, 2, 0, 0, 0} // Dynamic array using vector (recommended) vector<int> vec = {10, 20, 30}; vec.push_back(40); // Can grow dynamically

Java Arrays

Java
// Declaration int[] arr1 = new int[5]; // All elements initialized to 0 // Declaration with initialization int[] arr2 = {10, 20, 30, 40, 50}; // ArrayList (dynamic array) ArrayList<Integer> list = new ArrayList<>(); list.add(10); list.add(20); // Access elements int value = arr2[0]; // 10 int length = arr2.length; // 5

Python Lists (Dynamic Arrays)

Python
# List creation arr1 = [10, 20, 30, 40, 50] # Empty list arr2 = [] # List with specific size arr3 = [0] * 5 # [0, 0, 0, 0, 0] # List operations arr1.append(60) # Add element arr1.insert(0, 5) # Insert at index 0 arr1.pop() # Remove last element arr1.remove(20) # Remove specific value # Access elements value = arr1[0] # First element last = arr1[-1] # Last element length = len(arr1) # Length of list

🔧 Common Array Operations

1. Traversal - Visit All Elements

C++
void printArray(int arr[], int n) { for (int i = 0; i < n; i++) { cout << arr[i] << " "; } cout << endl; } // Time: O(n), Space: O(1)

2. Insertion - Add Element

C++
// Insert element at specific position void insertElement(int arr[], int& n, int pos, int value) { // Shift elements to the right for (int i = n; i > pos; i--) { arr[i] = arr[i-1]; } arr[pos] = value; n++; // Increase size } // Time: O(n) - worst case when inserting at beginning // Space: O(1)

3. Deletion - Remove Element

C++
// Delete element at specific position void deleteElement(int arr[], int& n, int pos) { // Shift elements to the left for (int i = pos; i < n-1; i++) { arr[i] = arr[i+1]; } n--; // Decrease size } // Time: O(n) - worst case when deleting from beginning // Space: O(1)

4. Searching - Find Element

C++
// Linear Search int linearSearch(int arr[], int n, int target) { for (int i = 0; i < n; i++) { if (arr[i] == target) { return i; // Return index } } return -1; // Not found } // Time: O(n), Space: O(1)

5. Reverse Array

C++
// Two-pointer approach void reverseArray(int arr[], int n) { int left = 0, right = n - 1; while (left < right) { swap(arr[left], arr[right]); left++; right--; } } // Time: O(n), Space: O(1)

⚡ Time & Space Complexity

Operation Time Complexity Space Complexity Notes
Access O(1) O(1) Direct index access
Search (Unsorted) O(n) O(1) Linear search required
Search (Sorted) O(log n) O(1) Binary search possible
Insert (End) O(1) O(1) If space available
Insert (Position) O(n) O(1) Requires shifting elements
Delete O(n) O(1) Requires shifting elements

🎨 Multi-Dimensional Arrays

Arrays can have multiple dimensions. 2D arrays (matrices) are most common, used in image processing, game boards, and dynamic programming.

C++
// 2D Array Declaration int matrix[3][4]; // 3 rows, 4 columns // 2D Array Initialization int mat[3][3] = { {1, 2, 3}, {4, 5, 6}, {7, 8, 9} }; // Accessing elements int value = mat[1][2]; // 6 (row 1, column 2) // Traversing 2D array for (int i = 0; i < 3; i++) { for (int j = 0; j < 3; j++) { cout << mat[i][j] << " "; } cout << endl; } // Time: O(rows × columns)

💡 Advantages & Disadvantages

✅ Advantages

  • • Fast Access: O(1) time to access any element by index
  • • Cache Friendly: Contiguous memory improves cache performance
  • • Simple: Easy to understand and implement
  • • Memory Efficient: No extra memory for pointers
  • • Predictable: Fixed size makes memory management easier

❌ Disadvantages

  • • Fixed Size: Cannot grow or shrink dynamically (static arrays)
  • • Insertion/Deletion Cost: O(n) time due to shifting elements
  • • Memory Waste: Unused allocated space if array not filled
  • • Homogeneous: Can only store one data type
  • • Contiguous Memory: Large arrays may fail to allocate

🎯 Common Interview Problems

1. Two Sum Problem

Easy

Find two numbers in array that add up to a target sum.

Input: arr = [2, 7, 11, 15], target = 9
Output: [0, 1] (arr[0] + arr[1] = 2 + 7 = 9)
Approach: Use hash map to store complements - O(n) time, O(n) space

2. Find Maximum Subarray Sum (Kadane's)

Medium

Find contiguous subarray with largest sum.

Input: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
Output: 6 (subarray [4, -1, 2, 1])
Approach: Dynamic programming - O(n) time, O(1) space

3. Rotate Array

Easy

Rotate array to the right by k steps.

Input: arr = [1,2,3,4,5,6,7], k = 3
Output: [5,6,7,1,2,3,4]
Approach: Reverse technique (3 reverses) - O(n) time, O(1) space

4. Merge Sorted Arrays

Easy

Merge two sorted arrays into one sorted array.

Input: arr1 = [1,3,5], arr2 = [2,4,6]
Output: [1,2,3,4,5,6]
Approach: Two pointers - O(n+m) time, O(n+m) space

🎓 Key Takeaways

  • ✓ Arrays provide O(1) access time - fastest for random access
  • ✓ Use arrays when you know the size beforehand or need fast access
  • ✓ Consider dynamic arrays (vector/ArrayList) for flexibility
  • ✓ Arrays are the foundation - master them before moving to complex structures
  • ✓ Many problems can be solved with arrays + two pointers or sliding window
